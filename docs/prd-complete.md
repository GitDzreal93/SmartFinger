# 灵巧手指·幽灵指环 完整版 PRD

## 1. 产品概述

### 1.1 产品名称

灵巧手指 / 幽灵指环

### 1.2 产品定位

一款基于 ESP32-C3 的外置电容屏自动点击设备，通过导电泡棉、人体接地铜箔、MOS/模拟开关和固件控制，在手机屏幕上实现可控、可调、可验证的自动点击。

产品重点不是单纯追求极限速度，而是实现稳定、可复现、可佩戴、可调节的屏幕点击辅助能力，适用于允许范围内的自动化测试、重复输入辅助、游戏离线挂机测试、无障碍辅助实验和硬件学习。

### 1.3 背景与问题

手机电容屏需要接近人体手指的电容变化才能触发点击。用户希望用低成本免焊硬件搭建一个可以快速验证的外置点击器，并在后续升级为更小、更稳定、更易用的指环式设备。

现有初稿已经明确了 ESP32-C3、D4184 MOS 模块、导电泡棉、铜箔胶带、微动开关和面包板方案，但仍需要补齐产品边界、硬件可行性验证、模式定义、参数调节、安全策略、异常处理和验收标准。

## 2. 目标与非目标

### 2.1 产品目标

1. 实现手机电容屏的稳定单次点击。
2. 支持手动按键触发、长按连点、模式切换和速度调节。
3. 支持免焊原型搭建，并预留可佩戴指环化升级路径。
4. 提供清晰的状态反馈，用户能知道当前是待机、单点、连点、定时还是异常状态。
5. 提供安全限速、默认关闭、断连保护和误触防护，降低误操作风险。
6. 输出可重复测试的验收方法，能判断设备是否真正稳定可用。

### 2.2 非目标

1. 不保证绕过 App 风控、验证码、排队系统或平台限制。
2. 不承诺在所有手机、所有贴膜、所有 App 页面上都能达到同样点击频率。
3. 不做侵入式手机改造，不接入手机内部电路。
4. 不在完整版第一阶段开发复杂手机 App；蓝牙控制可作为后续增强模块。
5. 不把设备定位为违反平台规则的抢购作弊工具。

## 3. 用户与使用场景

### 3.1 目标用户

1. 硬件初学者：希望用免焊方式完成一个可见、可测、可玩的 ESP32 项目。
2. 自动化测试用户：需要对手机页面上的固定位置做重复点击测试。
3. 重复输入辅助用户：在合规场景下减少手动重复点击。
4. 后续改装用户：希望把面包板原型升级为可佩戴、便携的小设备。

### 3.2 核心场景

1. 单次点击测试：用户按下按键，目标屏幕位置触发一次点击。
2. 长按连点：用户持续按住按键，设备按设定间隔连续点击，松开立即停止。
3. 定频连点：用户切换到连点模式后，设备持续按固定频率点击，再次操作停止。
4. 定时启动：用户设置倒计时或指定等待时间，到点后开始点击。
5. 多档速度：用户通过按键组合或旋钮在低、中、高速之间切换。
6. 佩戴使用：用户将铜箔贴在手指/手腕，点击头固定在屏幕或指环触点上。

## 4. 产品形态

### 4.1 完整版硬件组成

| 模块 | 建议器件 | 作用 | 备注 |
|---|---|---|---|
| 主控 | ESP32-C3 SuperMini | 固件运行、IO 控制、蓝牙预留 | 3.3V 逻辑 |
| 点击开关 | 小信号 MOS / D4184 模块 / CD4066 | 控制点击头接地或悬空 | 优先验证 D4184，稳定版建议小信号 MOS 或模拟开关 |
| 点击头 | 导电泡棉 | 接触手机屏幕 | 需要贴合稳定、面积适中 |
| 人体参考地 | 铜箔胶带 | 连接人体与系统 GND | 必须接触皮肤 |
| 输入 | 微动开关 x1-2 | 触发、切换模式、调速 | MVP 可只用 1 个 |
| 状态反馈 | LED / 板载 LED / 蜂鸣器可选 | 显示工作状态 | 完整版建议至少 LED |
| 供电 | USB-C / 小锂电池模块可选 | 供电 | 原型阶段先 USB 供电 |
| 结构 | 面包板 / 3D 打印外壳 / 指环固定件 | 原型和佩戴 | 完整版预留结构设计 |

### 4.2 电气原则

1. 点击头不主动向手机屏幕输出 3.3V。
2. 点击动作本质是让导电泡棉在“接近人体地/系统地”和“断开悬空”之间切换。
3. ESP32 只控制 MOS/模拟开关，不直接承担点击回路。
4. 铜箔、ESP32 GND、开关模块 GND 需要形成共同参考。
5. 所有接线必须先在断电状态完成，再通电测试。

## 5. 功能需求

### 5.1 FR-001 单次点击

用户短按微动开关时，设备对屏幕执行一次点击。

验收标准：

1. 短按一次，只产生一次点击事件。
2. 单次点击按下时长默认 40ms，可在固件常量中调整。
3. 连续短按 20 次，成功触发率不低于 90%。

### 5.2 FR-002 长按连点

用户按住微动开关超过长按阈值后，设备进入连点；松开按键后立即停止。

验收标准：

1. 长按阈值默认 500ms。
2. 松开按键后 100ms 内停止输出点击。
3. 默认连点间隔为 80ms，可配置为 40ms、80ms、120ms 三档。

### 5.3 FR-003 模式切换

设备支持至少三种模式：

1. 手动模式：短按单击，长按连点。
2. 锁定连点模式：短按开始，再次短按停止。
3. 定时模式：倒计时结束后自动开始点击。

验收标准：

1. 用户能通过按键组合或第二个按键切换模式。
2. 每次切换模式都有 LED 状态提示。
3. 上电默认进入手动模式，避免误启动。

### 5.4 FR-004 速度调节

设备支持点击频率调节。

建议档位：

| 档位 | 按下时长 | 抬起间隔 | 理论频率 | 用途 |
|---|---:|---:|---:|---|
| 慢速 | 60ms | 140ms | 5 次/秒 | 稳定性测试 |
| 中速 | 50ms | 70ms | 8 次/秒 | 常规重复点击 |
| 快速 | 40ms | 40ms | 12.5 次/秒 | 高频测试 |
| 极限 | 20ms | 20ms | 25 次/秒 | 仅用于实验 |

验收标准：

1. 设备至少支持慢速、中速、快速三档。
2. 当前档位可通过 LED 闪烁次数识别。
3. 极限档默认不作为上电默认值。

### 5.5 FR-005 状态反馈

设备通过 LED 表示状态。

| 状态 | LED 表现 |
|---|---|
| 待机 | 慢闪 |
| 单次点击 | 快闪一次 |
| 长按连点 | 高频闪烁 |
| 锁定连点 | 双闪循环 |
| 定时等待 | 呼吸或倒计时闪烁 |
| 异常/保护 | 快速三连闪 |

验收标准：

1. 用户不看串口也能判断设备是否正在点击。
2. 异常状态不会继续输出点击。

### 5.6 FR-006 参数持久化

完整版本支持保存用户选择的速度档位和默认模式。

验收标准：

1. 断电重启后保留上次速度档位。
2. 上电仍默认不自动开始点击。
3. 存储失败时回落到中速档。

### 5.7 FR-007 定时启动

用户可设置倒计时，设备倒计时结束后自动开始点击。

验收标准：

1. 支持 3 秒、5 秒、10 秒、30 秒四档倒计时。
2. 倒计时期间可按键取消。
3. 倒计时结束后进入锁定连点，再次按键停止。

### 5.8 FR-008 蓝牙控制预留

完整版本预留 BLE 控制能力，用于后续手机或网页控制面板。

第一阶段只需预留固件结构，不强制实现 App。

预留控制项：

1. 开始/停止。
2. 模式切换。
3. 速度档位。
4. 点击时长和间隔。
5. 设备状态读取。

## 6. 非功能需求

### 6.1 稳定性

1. 连续运行 10 分钟不死机。
2. 按键抖动不会造成误触发。
3. 点击输出状态不会卡在常开。

### 6.2 安全性

1. 默认上电不点击。
2. 软件设置最大连续点击时长，默认 60 秒后自动停止。
3. 点击头不输出高电平到屏幕。
4. USB 供电和电池供电方案都必须共地明确。
5. 异常状态下点击输出强制关闭。

### 6.3 可维护性

1. 固件使用清晰的状态机组织。
2. 引脚、时长、档位集中配置。
3. 点击输出、按键读取、模式控制、状态反馈拆分为独立模块或函数。

### 6.4 易用性

1. 新手可按接线表完成搭建。
2. 文档包含最小验证步骤。
3. 文档明确常见故障和排查路径。

## 7. 交互设计

### 7.1 单按键方案

| 操作 | 行为 |
|---|---|
| 短按 | 单击一次 |
| 长按 | 按住期间连点 |
| 双击 | 切换速度档位 |
| 三击 | 切换模式 |
| 长按 3 秒 | 停止并回到待机 |

### 7.2 双按键方案

| 按键 | 操作 | 行为 |
|---|---|---|
| 主按键 | 短按 | 单击或开始/停止 |
| 主按键 | 长按 | 临时连点 |
| 模式键 | 短按 | 切换模式 |
| 模式键 | 长按 | 切换速度 |

完整版本推荐双按键，降低误操作和学习成本。

## 8. 固件逻辑

### 8.1 状态机

设备包含以下状态：

1. `IDLE`：待机。
2. `SINGLE_CLICK`：执行单次点击。
3. `HOLD_REPEAT`：长按连点。
4. `LOCK_REPEAT`：锁定连点。
5. `COUNTDOWN`：定时等待。
6. `FAULT`：异常保护。

### 8.2 点击周期

一次点击包含两个阶段：

1. `PRESS`：MOS 导通，让点击头接近人体地。
2. `RELEASE`：MOS 断开，让点击头悬空。

### 8.3 按键处理

1. 必须做消抖，建议 20-30ms。
2. 短按、长按、双击需要基于时间窗口判断。
3. 长按连点优先级高于双击识别。

## 9. 硬件验证流程

### 9.1 最小电容触发验证

1. 将导电泡棉贴在屏幕目标位置。
2. 将铜箔贴在手指或手腕皮肤上。
3. 用杜邦线让泡棉短暂接触铜箔/GND，观察是否触发点击。
4. 若不能触发，调整泡棉面积、贴合压力、铜箔接触面积和手机贴膜条件。

### 9.2 MOS 开关验证

1. ESP32 输出低电平时，泡棉应处于悬空或非触发状态。
2. ESP32 输出高电平时，泡棉通过 MOS 接到 GND/人体参考地。
3. 用慢速档测试，确认单次点击稳定后再提高频率。

### 9.3 兼容性验证

至少测试：

1. 一台 iPhone 或 iPad。
2. 一台 Android 手机。
3. 有贴膜和无贴膜状态。
4. 不同点击头面积。
5. USB 供电和移动电源供电。

## 10. 验收标准

### 10.1 原型验收

1. 短按能触发单次点击。
2. 长按能持续连点，松手停止。
3. 三档速度可切换。
4. LED 能表示当前状态。
5. 连续运行 10 分钟无死机。
6. 默认上电不会自动点击。

### 10.2 完整版验收

1. 支持手动、锁定连点、定时三种模式。
2. 支持速度档位保存。
3. 支持最大连续点击时长保护。
4. 支持可佩戴固定方案，使用过程中点击头不明显位移。
5. 形成完整接线图、固件说明、故障排查表和测试记录。

## 11. 风险与应对

| 风险 | 表现 | 应对 |
|---|---|---|
| 电容屏不识别 | 点击无反应 | 增大泡棉面积、改善贴合、确保铜箔接触皮肤 |
| 一直长按不松 | 页面只识别按住 | 确保 MOS 有断开阶段，调大 release 间隔 |
| 点击过快丢事件 | App 无响应或漏点 | 降低频率，从 5-12 次/秒开始 |
| MOS 模块不适配 | 输出逻辑异常 | 换小信号 MOS 或 CD4066 |
| 误触发 | 上电即点击 | 默认关闭输出，增加启动延迟和状态机保护 |
| 平台限制 | 操作被风控 | 仅用于合规测试和辅助场景 |
