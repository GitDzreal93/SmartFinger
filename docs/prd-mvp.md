# 灵巧手指·幽灵指环 MVP PRD

## 1. 产品概述

### 1.1 MVP 名称

灵巧手指 MVP：旋钮调档屏幕点击验证台

### 1.2 MVP 目标

本阶段先不做手指佩戴、人体铜箔接地和指环形态，只做一个放在桌面上的调参验证原型：

1. ESP32-C3 作为主控。
2. 旋钮编码器作为输入，用来切换点击档位。
3. 0.96 寸 I2C OLED 显示当前档位、运行状态和频率。
4. 舵机带动导电海绵或触控头做真实的抬起/按下动作。
5. 先验证机械点击路径能否稳定连续触发手机屏幕。

MVP 的核心问题是：在电容开关方案尚不稳定、MOS 尚未到货或验证前，能否用舵机机械动作先跑通稳定连续点击，并通过旋钮直观看到和调整点击频率。

## 2. 成功标准

MVP 完成时必须满足：

1. 旋钮转动时，档位可以在 0-5 档之间切换。
2. 显示屏能实时显示当前档位，例如 `G-1`、`G-2`、`G-3`。
3. 档位为 0 时停止点击，档位为 1-5 时自动按对应频率点击。
4. 点击过程中，手机屏幕能收到连续点击事件。
5. 舵机能带动点击头完成真实抬起/按下动作，不依赖人体接触。
6. 上电默认停止点击，不自动开始。

## 3. MVP 范围

### 3.1 包含功能

1. 一个点击头，可以是导电泡棉、铜箔小片或触控笔头。
2. 一个三脚旋钮编码器模块，支持左旋、右旋。
3. 一个显示模块，用于显示档位。
4. 一个点击输出 GPIO。
5. 一个 SG90 类小舵机，用于带动点击头机械点击。
6. 5 个速度档位。
7. 旋钮切换档位。
8. 通过 `Grade 0` 停止，通过 `Grade 1-5` 自动连点。
9. 串口日志辅助调试。

### 3.2 不包含功能

1. 手指佩戴结构。
2. 人体铜箔接地。
3. 微动开关触发。
4. 短按单击/长按连点的复杂按键逻辑。
5. 蓝牙控制。
6. 手机 App。
7. 多点击头。
8. 定时启动。
9. 参数断电保存。
10. 电池供电。
11. 3D 打印外壳。

## 4. 硬件需求

### 4.1 必需硬件

| 硬件 | 数量 | 用途 |
|---|---:|---|
| ESP32-C3 SuperMini | 1 | 主控 |
| 三脚旋钮编码器模块 | 1 | 调整档位 |
| 0.96 寸 I2C OLED | 1 | 显示当前档位、运行状态和频率 |
| SG90 类小舵机 | 1 | 机械点击 |
| 导电泡棉/铜箔小片/触控笔头 | 1 | 点击头 |
| 面包板 | 1 | 免焊连接 |
| 杜邦线 | 若干 | 连接与传导 |
| USB-C 数据线 | 1 | 供电和烧录 |

### 4.2 推荐模块假设

从当前硬件确认看，MVP 暂按以下模块设计：

1. 旋钮编码器：三脚模块，引脚为 `A`、`C`、`B`。
2. 显示屏：0.96 寸 OLED IIC 模块，照片丝印为 `GND VCC SCL SDA`，常见驱动芯片为 SSD1306，常见 I2C 地址为 `0x3C`。

OLED 只需要 4 根线，比裸数码管更适合 MVP。第一版直接显示档位、运行状态和理论点击频率。

## 5. 接线原则

### 5.1 总体原则

1. MVP 不接人体铜箔。
2. MVP 不要求手指接触任何导电部分。
3. 点击头不接 3.3V，不向手机屏幕输出电压。
4. 点击动作通过舵机完成真实的物理抬起/按下。
5. 舵机供电优先使用 5V/VBUS 或外部 5V，不能接 ESP32 `3V3`。

### 5.2 推荐引脚

| ESP32-C3 引脚 | 连接对象 | 说明 |
|---|---|---|
| GPIO3 | 舵机信号线 | 舵机 PWM 控制 |
| GPIO4 | 编码器 `CLK` | 旋钮 A 相 |
| GPIO5 | 编码器 `DT` | 旋钮 B 相 |
| 3.3V | 编码器 `+` | 编码器供电 |
| GND | 编码器 `GND` | 编码器接地 |
| GPIO8 | OLED `SDA` | I2C 数据 |
| GPIO9 | OLED `SCL` | I2C 时钟 |
| 3.3V | OLED `VCC` | OLED 供电 |
| GND | OLED `GND` | OLED 接地 |
| 5V/VBUS | 舵机红线 | 舵机供电 |
| GND | 舵机棕/黑线 | 舵机接地 |

### 5.3 点击头形态

优先测试三种点击头：

1. 导电泡棉：固定在舵机摆臂末端，轻点屏幕。
2. 铜箔小片：固定在舵机摆臂末端，轻点屏幕。
3. 触控笔头：固定在舵机摆臂末端，轻点屏幕。

通过标准以“最稳定能触发的形态”为准。若杜邦线裸端不稳定，允许使用导电泡棉作为接触面积放大器，但仍不接人体。

## 6. 软件需求

### 6.1 引脚定义

| 名称 | 默认值 | 说明 |
|---|---:|---|
| `SERVO_PIN` | GPIO3 | 控制舵机 PWM |
| `ENC_CLK_PIN` | GPIO4 | 旋钮 A 相 |
| `ENC_DT_PIN` | GPIO5 | 旋钮 B 相 |
| `OLED_SDA_PIN` | GPIO8 | OLED I2C 数据 |
| `OLED_SCL_PIN` | GPIO9 | OLED I2C 时钟 |

### 6.2 档位定义

MVP 使用 5 档速度：

| 档位 | 显示 | 按下时长 | 抬起间隔 | 理论频率 | 用途 |
|---|---|---:|---:|---:|---|
| 0 | `Grade: 0` | 0ms | 0ms | 0 次/秒 | 停止 |
| 1 | `G-1` | 100ms | 250ms | 约 3 次/秒 | 最稳验证 |
| 2 | `G-2` | 90ms | 160ms | 约 4 次/秒 | 低速测试 |
| 3 | `G-3` | 80ms | 120ms | 约 5 次/秒 | 默认档 |
| 4 | `G-4` | 70ms | 90ms | 约 6 次/秒 | 高频测试 |
| 5 | `G-5` | 60ms | 70ms | 约 8 次/秒 | 极限实验 |

上电默认档位为 3 档，但上电默认不开始点击。

### 6.3 交互逻辑

| 用户操作 | 设备行为 |
|---|---|
| 上电 | 显示 `Grade: 0`，点击输出关闭 |
| 顺时针旋转 | 档位 +1，最高到 5 |
| 逆时针旋转 | 档位 -1，最低到 0 |
| 档位为 0 | 停止自动点击 |
| 档位为 1-5 | 自动按当前档位点击 |
| 点击运行中旋转 | 立即切换到新档位继续点击 |

### 6.4 显示逻辑

| 状态 | 显示内容 |
|---|---|
| 停止 | `Mode: STOP`、`Grade: 0`、`Rate: 0/s` |
| 运行中 | `Mode: RUN`、`Grade: 当前档位`、`Rate: 理论频率` |
| 旋钮调整 | 档位和频率实时刷新 |

OLED 第一版建议显示：

```text
SmartFinger
Mode: STOP
Grade: 0
Rate: 0/s
```

### 6.5 固件状态

MVP 只需要两个主状态：

1. `STOPPED`：`Grade 0`，停止点击。
2. `RUNNING`：`Grade 1-5`，按当前档位循环点击，旋钮可实时调整速度。

按键需要做 25ms 消抖，旋钮转动需要过滤抖动和反向毛刺。

## 7. 验收测试

### 7.1 旋钮与显示测试

步骤：

1. 上电后观察显示屏。
2. 顺时针旋转旋钮 5 次。
3. 逆时针旋转旋钮 5 次。
4. 将旋钮从 `Grade 0` 转到 `Grade 1`，再转回 `Grade 0`。

通过标准：

1. 上电显示默认档位。
2. 旋钮转动能稳定改变档位。
3. 档位不会超过 0-5。
4. `Grade 0` 显示停止，`Grade 1-5` 显示运行。
5. 运行状态在显示上有可见区别。

### 7.2 点击触发测试

步骤：

1. 打开手机计算器、备忘录或任意有明确点击反馈的页面。
2. 将舵机摆臂上的点击头对准目标按钮。
3. 确认 `Grade 0` 时点击头离开屏幕 1-3mm。
4. 将旋钮转到 `Grade 1` 开始点击。
5. 从 1 档开始测试，逐步升到 5 档。

通过标准：

1. 至少一个档位能稳定触发点击。
2. 停止后手机不再收到点击事件。
3. 运行中切换档位后，点击节奏能明显变化。

### 7.3 接触形态测试

分别测试：

1. 杜邦线裸端。
2. 铜箔小片。
3. 导电泡棉。

记录每种形态在 1-5 运行档下的成功率，选择最稳定的点击头作为后续版本基础。

## 8. 风险与处理

| 问题 | 可能原因 | 处理方式 |
|---|---|---|
| 完全点不动 | 点击头没有碰到屏幕或触控材料不合适 | 调整舵机角度、换导电泡棉或触控笔头 |
| 只识别长按 | 点击头没有真正抬起 | 调整 `SERVO_UP_US` 或安装高度 |
| 速度越快越不准 | 舵机动作跟不上或手机/App 丢事件 | 优先使用 1-3 档，记录稳定档位 |
| 旋钮乱跳 | 编码器抖动 | 增加软件滤波或使用成熟编码器库 |
| OLED 不亮 | 供电、SCL/SDA、I2C 地址或驱动库不匹配 | 确认 `GND/VCC/SCL/SDA`，运行 I2C 扫描，常见地址为 `0x3C` |
| ESP32 重启 | 舵机电流过大导致 USB 供电不足 | 舵机改用外部 5V，并与 ESP32 共地 |

## 9. MVP 开发顺序

1. 确认旋钮编码器和显示屏模块型号。
2. 点亮显示屏，显示默认档位。
3. 读取旋钮转动，实现 0-5 档切换。
4. 实现 `Grade 0` 停止、`Grade 1-5` 自动运行。
5. 接入舵机，实现固定频率机械点击输出。
6. 将档位映射到不同点击频率。
7. 将导电泡棉/触控笔头固定到舵机摆臂，测试真实手机触发。
8. 记录不同点击头和不同档位下的触发成功率。

## 10. MVP 交付物

1. 面包板接线图或接线照片。
2. ESP32-C3 Arduino 固件。
3. 旋钮档位和显示规则说明。
4. 点击头形态测试记录。
5. 1-5 档点击成功率记录。
6. 已知问题列表。

## 11. MVP 到完整版的升级路径

MVP 通过后，再进入完整版：

1. 增加手指/手腕铜箔人体参考地能力。
2. 增加短按单击、长按连点等手动触发模式。
3. 增加最大连续点击保护。
4. 增加锁定连点和定时启动。
5. 增加参数断电保存。
6. 小型化布线和可佩戴结构。
7. 预留 BLE 控制。
